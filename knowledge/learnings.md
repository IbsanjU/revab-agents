### 2026-07-09
- Self-review follow-up: framework now has real tests. `npm test` runs Node's built-in test runner (`node --import tsx --test`) over `utils/manifest.test.ts`, `orchestrator/queue.test.ts`, and `scripts/check-conventions.test.ts` (27 tests). Tests use env-var overrides (`MANIFEST_PATH_OVERRIDE`, `QUEUE_ROOT_DIR`) + cache-busted dynamic imports to isolate module state per test.
- `orchestrator/queue.ts` now has crash-recovery: `reclaimStale(staleMs)` requeues tasks stuck in `running` past a threshold (default 10min), tracked via `task.reclaimCount`; after `MAX_RECLAIMS` (3) it moves the task to `failed` instead of looping forever. Worker sweeps every `WORKER_STALE_SWEEP_MS` (default 60s); `npm run task -- reclaim [staleMs]` triggers it manually.
- Fixed a real pre-existing bug found while testing: `enqueue()` task ids could collide/misorder within the same millisecond (relied on `Date.now()-randomUUID` string sort for FIFO). Now includes a monotonic per-process sequence number in the id.
- Hardened `agents/registry.ts`'s `import-agents` handler: `payload.source` is now checked against a shell-metacharacter denylist and must resolve to an existing directory before being passed to `execCommand("npm", ...)` (which uses `spawn(..., { shell: true })` — argv entries there are not immune to injection).
- Added `npm run check:conventions` (`scripts/check-conventions.ts`): validates every `skills/*/SKILL.md` frontmatter (`name` matches dir, non-empty `description`) and that every `agents/registry.ts` handler calling `resolveProjectRepoPath(...)` first guards on `payload.project`.
- Added `npm run knowledge:rotate` (`scripts/rotate-learnings.ts`): archives all-but-current-month dated entries from `knowledge/learnings.md` into `knowledge/learnings/<YYYY-MM>.md` once the main file exceeds ~8KB; no-op below that threshold.
- Deferred: a dedicated OCR/speech-to-text MCP server (still the biggest documented capability gap) — not built this pass, flagged as the highest-leverage next server if image/video requirement extraction becomes a frequent real workflow.

### 2026-07-12 (conduct adaptations from reference prompts)
- Compared `.github/copilot-instructions.md`'s shared **Agent conduct** section against patterns (paraphrased, not copied) from a local reference collection of other coding-agent system prompts (`~/Documents/github/system_prompts_leaks`, outside this repo). Adopted 3 of 5 candidates: (1) a new universal **Faithful reporting** subsection — report outcomes exactly as observed, don't imply success by omission, prioritize accuracy over agreeableness (especially binding for reporter/researcher); (2) hard rule #10 now states approval is scoped to the previewed payload only — a prior "yes" never authorizes the same/similar write again later; (3) **Memory hygiene** now requires verifying a recalled entry's named file/selector/endpoint/flag still matches current state before acting on it. Now strips any `<!-- shared-conduct:vN -->` block (any version) through EOF before appending the new one — verified via `npm run agents:conduct` producing "replaced" (not "updated") for all 9 personas with no duplication.

### 2026-07-12 (removed the shared-conduct propagation mechanism entirely)
- User pushback: the per-persona `## Conduct` blocks (Boundaries, Faithful reporting, Completion checklist, Anti-hallucination, Memory hygiene, Verbosity) echoed into all 9 `.github/agents/*.agent.md` files were "unnecessary duplicate instructions." On inspection this was correct: `.github/copilot-instructions.md` loads automatically alongside every persona regardless of which agent is active, so every propagated block was 100% redundant with text already in its "Agent conduct" and "Per-agent boundaries" sections — the propagation added maintenance cost (marker-bump dance, the `apply-agent-conduct.ts` script, the duplication bug fixed earlier this session) without the model ever seeing new information.
- Removed entirely (reverting the intent of commits `e3a365b`/`ad0eff3`, not just simplifying): deleted `scripts/apply-agent-conduct.ts`, the `agents:conduct` npm script, and `check:conventions`' rule 4 (`checkAgentConduct`, both the marker check and its 4 tests in `scripts/check-conventions.test.ts`). All 9 persona files now end with a single one-line `## Conduct` pointer to `.github/copilot-instructions.md` instead of a duplicated block. `knowledge/conventions.md`/`knowledge/memory.md` updated to match; the persona-boundaries-list-parity idea was dropped along with the rest rather than kept standalone, per the user's "remove this" instruction.
- Verified: `npm test` (57/57, 4 fewer than before — the removed agent-conduct tests), `check:conventions` OK, `typecheck` clean.
- Lesson for future sessions: before propagating/duplicating any shared rule text into multiple files "for emphasis," check whether the reader (human or model) already sees the canonical source unconditionally — if so, duplication is pure maintenance liability, not redundancy-as-safety-net.
