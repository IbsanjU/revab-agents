# Project Memory

In-repo Copilot memory — version-controlled, shared across machines and sessions.
Agents read this at session start and append learnings via the artifacts `knowledge_append` MCP tool (which writes to `knowledge/learnings.md`) or by directly editing this file.

---

## Framework facts

- **Architecture**: `revab-agents` is framework-only. It has no `tests/` and never runs Playwright/Cucumber/Allure against itself. Every test-authoring/execution/reporting operation targets a **project** declared in `projects/manifest.json` (index) + `projects/<name>/project.json` (per-project config; legacy single-file `projects.manifest.json` still supported as fallback), resolved via `utils/manifest.ts` (the trust boundary — never accept a raw `repoPath`/`repoUrl` from a payload/tool arg).
- **MCP servers** (local HTTP, stateless, `.vscode/mcp.json`):
  - `jira` :7311 — `jira_search`, `jira_get_issue`, `jira_get_epic_children`, `jira_add_comment`, `jira_create_issue` (dryRun default), `jira_bulk_create_issues` (dryRun default; per-row dedup search + partial-failure-tolerant batch create; each draft may carry a `transitionName` applied right after a successful, non-duplicate create), `jira_search_users` (resolve name/email -> accountId, optionally scoped to a project), `jira_assign_issue` (dryRun default; requires a resolved accountId, never a raw name), `jira_update_issue` (dryRun default), `jira_bulk_update_issues` (dryRun default; per-row `fields` update and/or `transitionName`, partial-failure tolerant — same transition-resolution as `jira_transition_issue`), `jira_transition_issue` (dryRun default), `jira_get_boards`/`jira_get_sprints`/`jira_get_backlog` (Agile REST reads), `jira_move_to_sprint` (dryRun default), `jira_get_sprint_report` (status/unassigned/missing-field counts for a sprint), `jira_save_issue` (asks for a project folder before saving locally) — `jira_delete_issue`'s implementation remains in source but its `registerTool` call is commented out (not registered), so the running server cannot delete issues; see `knowledge/conventions.md`. Transition lookup/apply logic is shared via `findTransition`/`applyTransition` helpers in `mcp-servers/jira/index.ts` (used by `jira_transition_issue`, `jira_bulk_create_issues`, and `jira_bulk_update_issues`).
  - `confluence` :7312 — `confluence_search`, `confluence_get_page` (format: text/html/structured — structured expands accordion/expand/tabs macros instead of flattening them), `confluence_get_children`, `confluence_get_attachments`, `confluence_download_attachment`, `confluence_get_comments`, `confluence_extract_links`, `confluence_create_page` (dryRun default), `confluence_update_page` (dryRun default), `confluence_add_comment` (dryRun default), `confluence_delete_page` (dryRun default), `confluence_save_page` (asks for a project folder before saving HTML+structured+metadata locally), `confluence_upload_attachment` (dryRun default — attach local images/diagrams to a page, returns an <ac:image> embed hint). Every URL returned by `confluence_search`/`confluence_save_page`/`confluence_extract_links` is normalized via `normalizeConfluenceUrl` (same logic duplicated in `scripts/confluence-sync-project.ts`) to keep the full space-qualified `/spaces/<SPACE>/pages/<id>/<Title>` form — the bare `/pages/<id>` form doesn't resolve in a browser without the space key. For a whole page tree (not just one page), `npm run confluence:sync -- --project <name> --rootPageId <id>` (`scripts/confluence-sync-project.ts`) crawls descendants + same-space content links, writes `${pageId}.html`/`.md`/`.json` per page plus attachment/link/media indexes under `projects/<project>/downloads/confluence/`, and is incremental by version (`--full` forces a re-crawl).
  - `jtmf` :7313 — `jtmf_get_test_case`, `jtmf_search_tests`, `jtmf_get_test_plan`, `jtmf_create_test_case` (dryRun default), `jtmf_update_test_case` (dryRun default), `jtmf_delete_test_case` (dryRun default), `jtmf_raw_get`
  - `github` :7320 — `github_search_code`, `github_search_repos`, `github_search_issues`, `github_search_commits`, `github_search_topics` (scope org|public — public results must be labeled external), `github_get_file`, `github_save_file` (ask-before-save, downloads/github/<project>/) — search tools read-only, repo-agnostic (org/repo are call-time qualifiers, not manifest-resolved); defaults to `GITHUB_ORG` when a call doesn't name its own org/repo
  - `artifacts` :7314 — `list_files`, `read_repo_file`, `knowledge_append`, `knowledge_search` (greps knowledge/** and projects/**/*.md for prior work) — scoped to this repo only
  - `media` :7319 — `get_file_metadata` (image/PDF/DOCX/generic metadata as JSON), `read_pdf_text`, `read_docx_text`, `read_excel_rows`/`read_csv_rows` (xlsx/xls/csv -> structured rows keyed by header row, for bulk ticket intake), `create_pdf`, `create_docx` (styled, accent-color support), `media_extract_requirements` (single fallback extractor when native vision is unavailable: auto-routes a file/directory by type — images/scanned PDFs to OCR, text PDFs/DOCX/XLSX/CSV to their parsers, diagrams to `read_diagram`, everything else to a plain-text fallback — and writes one consolidated local markdown report; `scripts/batch-media-extract.ts` / `npm run media:batch-extract -- --project <name>` calls it once per subfolder for trees larger than its per-call file cap, then merges the reports) — resolves paths against this repo by default or a manifest `project`'s repoPath if given; plus `ocr_image` (local tesseract.js OCR: text + blocks with bbox/confidence), `ocr_pdf` (page-by-page OCR of scanned PDFs via pdf-parse getScreenshot), `read_diagram` (draw.io XML / Mermaid / PlantUML -> { nodes, edges } graph via utils/diagramParse.ts; raster -> use ocr_image, best-effort), `create_diagram` (Mermaid -> SVG/PNG via npx @mermaid-js/mermaid-cli); uses `pdf-parse`, `mammoth`, `image-size`, `file-type`, `pdfkit`, `docx`, `tesseract.js`, `xlsx` (installed from SheetJS's own CDN tarball, not the npm registry — the npm-published `xlsx` build has unpatched prototype-pollution/ReDoS advisories; see `package.json`'s `xlsx` dependency)
  - `notify` :7321 — `notify_teams` (Adaptive Card to TEAMS_WEBHOOK_URL), `notify_email` (Outlook via Microsoft Graph GRAPH_* or smtp.office365.com SMTP_* fallback, nodemailer) — both dryRun default; worker sends automatic completion notices only for tasks that opt in via payload `notify` ("teams" or {channel:"email",to:[...]}, NOTIFY_EMAIL_TO fallback)
  - `playwright-runner` :7316 — `run_bdd`, `run_playwright`, `get_test_files` — all take `project`, execute in that project's resolved repoPath
  - `allure-report` :7317 — `generate_report`, `allure_summary`, `get_result_json` — all take `project`
  - `codegen` :7318 — `scaffold_feature` (requires `source` citation), `scaffold_step`, `scaffold_page`, `detect_conventions` — writes into the target project's `testPaths`
  - `playwright` :7315 — official `@playwright/mcp` via `npm run serve:playwright` (npx, local — target-agnostic interactive browser automation)
  - Shared infra: `mcp-servers/shared/server.ts` (`startMcpHttpServer`, `textResult`, `errorResult`), `mcp-servers/shared/http.ts` (`apiGet`, `apiPost`, `apiPut`, `apiDelete`, `apiGetBinary`, `authHeaders`, plus exported `buildUrl`/`handleJson` reused by other HTTP-backed servers — auth is per-service via `setAuthService("jira"|"confluence"|"jtmf")`, called once at each server's startup; `authHeaders()`/`apiGet`/etc. take no service argument, they read whichever service that process declared, preferring `<SERVICE>_EMAIL`/`<SERVICE>_API_TOKEN`/`<SERVICE>_AUTH_MODE` and falling back to the shared `ATLASSIAN_*` vars), `mcp-servers/shared/githubHttp.ts` (`githubGet`, GITHUB_TOKEN-based auth, `defaultGithubOrg`), `utils/fsSafety.ts` (`resolveWithinRoot`, the reusable path-containment check), `utils/saveSuggestion.ts` (`suggestProjectFolder`/`buildSaveConfirmationPrompt` for the ask-before-save pattern), `utils/confluenceMacros.ts` (`expandConfluenceMacros`), `utils/jiraLinks.ts` (`buildJiraIssueUrl`, shared by jira/jtmf servers), `utils/notify.ts` (sendTeams/sendEmail/buildTeamsPayload, shared by notify server + worker), `utils/diagramParse.ts` (parseDiagram/parseDrawio/parseMermaid/parsePlantuml). All MCP servers bind to 127.0.0.1; optional MCP_SHARED_SECRET enforces an x-mcp-secret header
- **Manifest**: `projects/` dir (zod-validated by `utils/manifest.ts`): `projects/manifest.json` index + `projects/<name>/project.json` per project, with per-project artifacts co-located (`app-model.md`, `downloads/`, `reports/`, `test-plans/`; `resolveProjectArtifactsDir()`); legacy root `projects.manifest.json` still works as fallback. Each project.json: `name`, `repoPath`/`repoUrl`+`branch`, `testPaths`, `jira`/`confluence`/`jtmf` ids, `execution.mode` (`local`|`browserstack`|`custom`, defaults `local` — BrowserStack only if detected in the target repo, never assumed).
- **Orchestrator**: file queue `.queue/pending|running|done|failed`; handlers in `agents/registry.ts` — `run-bdd`/`generate-report` require `payload.project`; `typecheck` stays scoped to this framework repo; CLI `npm run task -- enqueue|status|types|reclaim`; worker `npm run worker` — per-task timeout (WORKER_TASK_TIMEOUT_MS, default 30min), one opt-in retry (`payload.retryOnFailure: true`), opt-in completion notifications (`payload.notify`), and plan traceability (`payload.plan` = knowledge/plans path). Worker auto-sweeps stale `running` tasks (crash recovery) via `reclaimStale()`; queue root and manifest path are env-overridable (`QUEUE_ROOT_DIR`, `MANIFEST_PATH_OVERRIDE`) for tests.
- **Agents**: 10 files in `.github/agents/*.agent.md` (planner, orchestrator, researcher, test-planner, automation, reporter, documenter, importer, self-improve, bsa) — all operate on a manifest-resolved `project`. Planner-first is hard rule 13: non-trivial work needs a finalized, user-approved plan in `knowledge/plans/<project>/` before execution; researcher is strictly read-only (suggest-then-pull via ask-before-save tools incl. `github_save_file`). `bsa` is the entry point for requirements arriving as chat/Excel/CSV/docs rather than existing Jira tickets — parses input, drafts tickets to a required-fields template, routes assignees via `projects/<project>/team-roster.json`, bulk-creates (dryRun-first), and reports sprint/backlog health; never deletes (see jira_delete_issue note above).
- **Skills**: `skills/*/SKILL.md` — `analyze-test-failures`, `detect-execution-convention`, `upload-to-jtmf`, `update-jira-epic`, `extract-requirements-from-image`, `extract-requirements-from-video`, `consolidate-project-report`, `build-test-plan-interactive`, `search-across-sources` (federates Jira/Confluence/JTMF/GitHub topic search), `bulk-create-tickets`, `route-assignee`, `sprint-backlog-report`.
- **Knowledge**: `knowledge/learnings.md` (dated session entries) + `knowledge/conventions.md` (team rules) + `knowledge/app-model/<project>.md` (living app maps) + `knowledge/reports/<project>/<date>.md` (consolidated project reports) + `knowledge/plans/<project>/<date>-<slug>.md` (planner-approved plans). New per-project artifacts should prefer `projects/<name>/` (app-model.md, downloads/, reports/, test-plans/).
- **Tool names** (VS Code agent YAML): `edit/editFiles` (not `editFiles`), `runCommands`, `runTasks`, `codebase`, `search`, `problems`, `testFailure`, `fetch`.
- **Anti-hallucination rule**: every generated test case/script/Jira/JTMF write must carry a source citation (Jira key, Confluence page id, transcript timestamp, or app-model reference). Missing citation blocks generation — ask, don't invent.
- **Media limits**: OCR and diagram reading are covered (`ocr_image`, `ocr_pdf`, `read_diagram`); speech-to-text is deliberately out of scope — `extract-requirements-from-video` still relies on manually supplied transcripts. Confluence viewer analytics needs Cloud Premium.
- **Ask before saving locally**: `confluence_save_page`/`jira_save_issue` (and `confluence_download_attachment`'s optional `project`) never write to disk until a `project` folder is confirmed — if omitted they return a suggested default (`<KEY>-XXX`) and a prompt to ask the user first.
- **Windows PATH fix** per terminal session: `$env:Path = "C:\Program Files\nodejs;$env:Path"; Set-ExecutionPolicy -Scope Process Bypass -Force`.
- **Testing/tooling**: `npm test` (Node built-in test runner via `tsx`, covers `utils/manifest.ts`, `utils/diagramParse.ts`, `orchestrator/queue.ts`, `scripts/check-conventions.ts`), `npm run check:conventions` (skill frontmatter + registry `payload.project` guard + docs-drift lint: every registerTool name must appear in README.md and knowledge/memory.md, every server in .vscode/mcp.json), CI at `.github/workflows/ci.yml` (typecheck+test+conventions on push/PR), `npm run knowledge:rotate` (archives old `knowledge/learnings.md` entries into `knowledge/learnings/<YYYY-MM>.md` past ~8KB).
